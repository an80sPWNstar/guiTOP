## Project
guiTOP v0.2.2 — Electron desktop GPU monitor (local + multi-host SSH). Three skins: bars, gauges, corvette. Process table. Electron 31. Electron-builder for NSIS Windows exe. No build system — plain JS require().

### What's done

## How the wiring works

```
main.js
  └─ startClaudeSwap(callback) → polls cswap, calls callback(payload)
       └─ win.webContents.send('claude-swap', payload)  // main→renderer IPC

preload.js
  └─ onClaudeSwap: (cb) => ipcRenderer.on('claude-swap', cb)

renderer/renderer.js
  └─ window.guiTOP.onClaudeSwap(payload => { state.claudeSwap = payload; renderClaudeStrip() })

claude-usage-strip.js
  └─ ClaudeUsageStrip.update(root, usageData, skin, swapData)
       └─ updateSwap() → renders account chips + auto dot
```

Data shape from swap collector:
```js
{
  ok: true,
  ts: 1234567890,
  autoOn: false,        // ← THE GAP
  autoSinceMin: null,   // ← THE GAP
  accounts: [
    { number: 1, alias: 'bryan', active: true, disabled: false, fiveHourPct: 62, sevenDayPct: 41 },
    ...
  ]
}
```

## Next step: implement auto-daemon detection

In `src/collectors/claude-swap.js`:

1. Add a second poll loop that runs `cswap auto --json` (or check if it's running via process list)
2. Parse the JSON-line output — track the last `switch` event
3. Set `autoOn: true` when daemon is running, `autoSinceMin` = minutes since last switch event
4. Wire into the existing `onData` callback alongside the account list data

The spec from README:
> `cswap auto --json` emits one JSON event per line (`poll`, `switch`, `no-switch`, `account-quarantined`, `all-exhausted`, `error`). Track the timestamp of the last `switch` event for the `ON · xxM` readout. If the user doesn't run `cswap auto`, show `OFF`.

Also consider: `cswap auto` may not be running. The collector should degrade gracefully — if the command fails or returns nothing, `autoOn: false` is correct (already the default).
